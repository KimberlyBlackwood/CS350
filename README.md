Summarize the project and what problem it was solving.
This project required the use of different peripherals that were placed on the breadboard. I then built a working thermostat by coding the board to show changes in temperature. This resulted in the LED being activated and a value to update to stimulate real-time fluctuations in temperature.

What did you do particularly well?
Wow, I struggled with the breadboard through out this course. I seek help from tutors, family members and the instructor. In all I think I did well on creating variables, functions  and structuring the code. The comments that were provided was very easy to follow and understand.
Where could you improve?
I could improve by first being patient and trusting the process. I have a tendency to become flustered and feeling dejected when things are not going as planned. This happens especially when I am seeking help from many sources with little to no result. Eventually it all came together, and I realize that I need to really be patient when potting hardware together.
What tools and/or resources are you adding to your support network?
I  will definitely utilize reviewing the discussions posted in the course. Many of my peers posted questions and received answers to issues that were similar to what I was going through in the course. I will also reach out to the professor first before going to tutors.
What skills from this project will be particularly transferable to other projects and/or course work?
Seeing all the parts on the breadboard coming together and working as intended is a skill that can be transferred to other projects in the future. I learned how to code peripherals to work with each other by  combining button presses with custom variables while working with temperature sensors. It has now piqued  my interest to create a home security system that I found out that can be done through my research on Raspberry Pi.
How did you make this project maintainable, readable, and adaptable?
To  make this project maintainable, readable, and adaptable I added comments that explained what was going on in the code, variable and functionality which kept it logical. To see how each check of the timer work a while loop was used, as a result making it easy to make  changes.
